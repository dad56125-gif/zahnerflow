# ZahnerFlow Agent Guide

本文件是 ZahnerFlow 仓库内的项目级代理约束，适用于 Codex 及其他代码代理。
它只描述本仓库的工作规则，不描述你的全局个人偏好。

## 1. 任务开始前必读

每次开始任务前，先读取以下文件：

- `.memory/rules.md`
- `.memory/design.md`

如果任务涉及架构、运行拓扑、数据流、接口契约、设备行为、启动行为或持久化方式，再同时参考：

- `.memory/changelog.md`
- `GEMINI.md`

## 2. Python 与环境

- 使用 `uv` 管理 Python 环境、依赖和命令执行。
- 不要为本项目切换到其他 Python 包管理方式。
- 运行 Python 相关命令时，优先使用 `uv run ...`。
- Git 提交信息必须使用中文。

## 3. 当前项目架构事实

- 当前后端是单进程 Python 运行时，不是多服务架构。
- `AppRuntime` 是进程内唯一运行时协调器。
- SQLite 是本地持久化边界。
- 真机驱动由 Python 类直接导入调用，不作为独立服务启动。
- 禁止重新引入旧 worker IPC、额外设备服务端口或遗留 NestJS 后端作为当前运行时一部分。

如需精确表述，以 `.memory/design.md` 为准。

## 4. 修改代码时的强制规则

- 先核对真实代码、真实路由、真实端口、真实返回值，再做判断。
- 不要根据旧备忘、想象中的架构或名义分层直接下结论。
- 前端与后端契约变更后，先同步共享 contracts，再检查类型与构建是否通过。
- 前端通信入口以 `apps/frontend/src/runtimeClient.ts` 为准，不要重新分叉新的通信主路径。
- 如无明确设计记录，禁止加入兼容壳、过渡层或回退路径。

## 5. 验证约定

- 前端契约相关修改后，优先验证 shared contracts、`packages/types` 和前端构建。
- 后端测试从 `apps/python_backend` 上下文运行，必要时使用 `PYTHONPATH=.`。
- 与运行状态相关的问题，优先检查真实端口、真实快照、真实接口载荷。

## 6. 文档同步规则

如果修改影响以下任一内容，必须同步更新 `.memory/design.md` 与 `.memory/changelog.md`：

- 架构
- 运行拓扑
- 数据流
- 接口契约
- 设备行为
- 启动方式
- 持久化方式

要求：

- `changelog.md` 中使用的锚点必须存在于 `design.md`
- 不要保留与当前代码矛盾的旧描述

## 7. 并行与代理协作

- 任务可以拆分并行时，可以启动 Claude Code 处理可独立的子任务。
- 但最终项目事实、设计一致性和交付结果仍以本仓库当前代码与文档为准。

## 8. 项目内优先级

当多份说明发生冲突时，按以下顺序处理：

1. 当前代码与运行事实
2. `.memory/design.md`
3. `.memory/rules.md`
4. 本文件 `AGENTS.md`
5. `GEMINI.md`

如果仍有冲突，不要猜测，先以代码和运行结果为准，再修正文档。
