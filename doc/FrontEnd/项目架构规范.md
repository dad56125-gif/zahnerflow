前端开发指导
1. 目标与范围

目标：统一布局/样式/响应式策略，减少返工，提升可维护性。

范围：样式系统（设计令牌、单位、层级）、布局约束、可访问性、性能预算、评审与验收。

2. 术语与默认值

设计令牌（Design Tokens）：通过 CSS 变量定义尺寸/颜色/层级等。

单位策略：布局尺寸用 rem 与 clamp()；仅边框宽度/细分阴影允许 px。

容器查询：组件自适应优先用 @container，全局断点仅兜底。

滚动归属：内容区滚动，导航固定。

3. 文档限制（强制规则）

尺寸与单位

必须使用设计令牌：var(--space), --navbar-h, --sidebar-w 等。

禁止在布局中直接写裸 px（边框、发光阴影除外）。

关键尺寸采用：--xxx: clamp(min, preferred, max);

布局

固定头部：使用 position: fixed; height: var(--navbar-h);

主容器必须为头部预留：padding-top: calc(var(--navbar-h) + env(safe-area-inset-top, 0px));

视口高度使用 100dvh / min-block-size: 100dvh，禁止 calc(100vh - Npx) 硬扣。

响应式

优先容器查询：父容器声明 container-type: inline-size;

仅在版式切换需要时使用有限媒体断点（≤ 2 个）。

命名与层级

统一 BEM/语义命名：.top-navbar, .main-content, .sidebar。

z-index 只用令牌：--z-navbar, --z-overlay，禁止魔法数字。

可访问性

字体大小随 rem 放大；交互控件可触达高度 >= var(--btn-h)。

颜色对比满足 WCAG AA；为动效提供 prefers-reduced-motion 降级。

性能预算

首屏 CSS ≤ 150KB（压缩后），关键路径样式内联 ≤ 20KB。

避免在大面积区域使用高成本滤镜；backdrop-filter 需降级方案。

4. 设计令牌（示例，直接抄用）
:root {
  /* 尺寸基线（16px） */
  --space:    clamp(1rem, 0.8rem + 0.6vw, 1.75rem);
  --navbar-h: clamp(3rem, 2.5rem + 1vw, 4rem);       /* 48–64px */
  --sidebar-w: clamp(16rem, 20vw, 22rem);            /* 256–352px */

  /* 字体与控件 */
  --text-body:  clamp(0.95rem, 0.9rem + 0.2vw, 1.0625rem);
  --text-title: clamp(1.25rem, 1rem + 1vw, 2rem);
  --btn-h:      clamp(2.25rem, 2rem + 0.5vw, 2.75rem);

  /* 圆角/模糊/层级（已有可复用） */
  --radius-sm: 15px; --radius: 20px; --radius-lg: 25px;
  --blur-xl: 24px;
  --z-navbar: 40; --z-overlay: 50;
}

5. 布局基线（TopNavbar + 主容器）
.top-navbar {
  position: fixed; inset: 0 0 auto 0;
  height: var(--navbar-h);
  display: flex; align-items: center; justify-content: space-between;
  padding-inline: var(--space);
  z-index: var(--z-navbar);
  /* 玻璃、阴影、边框：沿用项目既有变量 */
}

.app-root {
  display: flex; flex-direction: column;
  padding-top: calc(var(--navbar-h) + env(safe-area-inset-top, 0px));
}

.main-content {
  container-type: inline-size;
  display: grid;
  grid-template-columns: minmax(0, 1fr);
  gap: var(--space);
  padding: var(--space);
  min-block-size: 100dvh; /* 避免移动端地址栏抖动 */
}

.main-content.has-sidebar {
  /* 由容器查询或兜底断点切换多栏 */
}
@container (min-width: 900px) {
  .main-content.has-sidebar { grid-template-columns: var(--sidebar-w) 1fr; }
}

6. 组件开发约定

组件入口容器加 container-type: inline-size; 以启用容器查询。

尺寸/间距只读令牌；新增需求先扩令牌再使用。

滚动放在内容区（如 .content { overflow:auto; }），避免根层滚动。

7. 代码评审清单（PR 必过）

 使用了设计令牌，未出现裸 px（边框/阴影除外）

 关键尺寸使用 clamp() 或 rem，未写死 calc(100vh - Npx)

 容器启用了 container-type；响应式优先容器查询

 z-index 使用令牌，无魔法数字

 可访问性（对比度、焦点、键盘）通过

 体积与性能预算未超标（CSS 拆分合理）

8. 验收标准（QA 样例）

分辨率：1366×768、1920×1080、2560×1440、3840×2160 下版式比例合理；

放大：浏览器缩放 125%/150% 布局不破；

移动端：地址栏收起/展开无跳动；

导航：固定头部不遮挡内容，内容滚动流畅（无抖动）；

侧栏：随容器宽度在 clamp 范围内流式变化。

## CSS 设计规范

### 1. 结构与模块化

为了提高可维护性和可重用性，`glass-ui.css` 将被拆分为多个模块化的文件。所有 CSS 文件将存放在 `src/styles/` 目录下。

**文件结构:**

```
src/
└── styles/
    ├── base/
    │   ├── _reset.css      # 重置浏览器默认样式
    │   ├── _variables.css  # CSS 变量 (颜色, 字体, 间距等)
    │   ├── _typography.css # 基础排版样式
    │   └── _utilities.css  # 通用工具类
    ├── components/
    │   ├── _button.css
    │   ├── _input.css
    │   ├── _device-hover.css # 设备悬浮提示框样式
    │   └── ...             # 其他组件样式
    ├── layout/
    │   ├── _grid.css
    │   ├── _header.css
    │   ├── _sidebar.css
    │   └── _footer.css
    ├── themes/
    │   ├── _dark.css
    │   └── _light.css
    └── main.css            # 主入口文件，用于 @import 所有模块
```

### 2. 命名规范

- **BEM (Block, Element, Modifier)**: 采用 BEM 命名约定来确保类名的清晰和无冲突。
  - `block`: 代表独立的组件 (e.g., `.card`, `.nav`)。
  - `element`: 代表 `block` 的一部分 (e.g., `.card__title`, `.nav__item`)。
  - `modifier`: 代表 `block` 或 `element` 的不同状态或变体 (e.g., `.card--dark`, `.nav__item--active`)。

### 3. CSS 变量

- **集中管理**: 所有全局颜色、字体、间距、断点等应定义为 CSS 变量，并存放在 `base/_variables.css` 中。
- **命名**: 变量名应清晰明了，例如 `--primary-color`, `--font-size-large`。

### 4. 文件规范

- **主入口**: `main.css` 是唯一的 CSS 入口文件，通过 `@import` 规则引入其他所有模块。
- **模块化**: 每个组件或布局单元都应该有自己的 CSS 文件。
- **下划线前缀**: 分部文件（partials）使用下划线前缀 (e.g., `_reset.css`)，表示它们是模块，应由 `main.css` 导入。
- **注释**: 在每个文件的开头添加注释，说明其用途。对于复杂的样式规则，也应添加简短注释。

**示例 `main.css`:**

```css
/* Base styles */
@import "base/_reset.css";
@import "base/_variables.css";
@import "base/_typography.css";
@import "base/_utilities.css";

/* Layout styles */
@import "layout/_grid.css";
@import "layout/_header.css";

/* Component styles */
@import "components/_button.css";
@import "components/_input.css";
```

## CSS 样式优化规范

### 1. 核心原则
- **无视觉回归**: 优化的首要目标是提升代码质量和性能，而不是改变现有设计。任何修改都不能导致视觉上的变化。
- **提升可维护性**: 代码应更易于理解、修改和扩展。
- **减少冗余**: 清理不必要的代码，减小文件体积。

### 2. 优化清单
- **重复内容 (Don't Repeat Yourself)**:
  - **识别**: 查找完全相同或高度相似的样式块。
  - **合并**: 将重复的样式抽象为通用的工具类（utility class）或合并到共享的组件样式中。
- **未被应用的样式 (Dead Code)**:
  - **识别**: 查找在整个项目中从未被使用过的 CSS 类名、ID 或选择器。
  - **移除**: 确认无用后，安全地从样式表中删除。
- **冲突与覆盖 (Specificity & Overrides)**:
  - **识别**: 分析样式被频繁覆盖的规则，这通常意味着选择器特异性（specificity）过高或结构混乱。
  - **简化**: 尽可能降低选择器的特异性，避免使用 `!important`，利用 CSS 的层叠性（cascade）而非强制覆盖。
- **精简内容 (Shorthands & Simplification)**:
  - **使用缩写**: 尽可能使用 CSS 缩写属性，例如用 `margin: 10px 20px;` 代替 `margin-top: 10px; margin-right: 20px; ...`。
  - **合并选择器**: 如果多个选择器共享相同的样式规则，将它们分组，例如 `h1, h2, h3 { font-weight: bold; }`。
- **文件与结构**:
  - **遵循现有规范**: 所有优化都必须在 `src/styles/` 的模块化结构内进行。
  - **按需拆分/合并**: 对于过大的文件，考虑按功能拆分。对于功能单一的小文件，考虑合并到相关模块中。
