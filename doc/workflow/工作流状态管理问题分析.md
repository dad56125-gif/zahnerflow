# 工作流状态管理问题分析

## 问题描述

工作流ID/名称显示功能不激活，历史工作流加载后状态不同步，历史记录排序混乱。

## 解决方案核心要点

1. **状态同步修复**：在历史工作流加载时同步更新WorkflowStore状态
2. **时间排序实现**：按创建时间降序排列历史工作流记录
3. **架构一致性**：确保CanvasStore和WorkflowStore状态管理的一致性

## 设计思路

### 状态管理架构分离
- **CanvasStore**：管理画布创作状态（节点、连接关系）
- **WorkflowStore**：管理工作流管理状态（选中的历史工作流）
- **同步机制**：确保两个状态系统在关键操作时保持一致

### 工作流生命周期优化
- **新建流程**：清空画布，准备创作环境
- **运行流程**：创建工作流到后端，持久化当前画布状态
- **加载历史**：同时恢复画布内容和工作流管理状态

## 关键决策

1. **选择在WorkflowManagerUI中修复**：作为历史工作流管理的核心组件，承担状态同步责任
2. **使用setCurrentWorkflow方法**：利用现有的WorkflowStore方法进行状态更新
3. **前端排序实现**：在数据处理层添加排序逻辑，保证用户体验一致性
4. **最小化修改范围**：集中修改WorkflowManagerUI.tsx，降低引入新问题的风险
