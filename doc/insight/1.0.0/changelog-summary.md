# 1.0.0 Changelog 汇总

## 最终包含的主要功能

1. 单进程 Python 后端运行时和 `AppRuntime` 协调器。
2. Furnace、MFC、Zahner 直接设备驱动和模拟器连接参数路由。
3. 统一设备 runtime 状态 envelope、REST 状态路由和 Socket.IO 推送。
4. 本地工作流执行状态机，包括运行、暂停、取消、刷新接管和运行前元数据确认。
5. 后端事实驱动 ETA、执行前估算、节点耗时学习和执行事实记录。
6. 循环展开、迭代事件、工作流块展开、高级节点展开和从展开步骤启动。
7. 自动 `startup` / `shutdown` 测量边界。
8. 工作流 fingerprint 自动归档、相似特征和实验地图。
9. 实验记录浏览器：工作流列表、定义视图、执行报告、收藏、重命名和地图视图。
10. Furnace 历史活动概览、日内曲线和 CSV 导出。
11. 前端 BEM、设计令牌、按钮、输入、下拉、浮层和通信入口收敛。
12. Electron 桌面壳与 Python 后端打包链路。

## Changelog 整理结果

`.memory/changelog.md` 已压缩为设计变更索引；压缩前的完整历史条目保存在 `changelog-full-history.md`。归档时做以下解释：

1. 完整历史中的所有条目都视为 1.0.0 完成过程的历史记录，不再作为待执行计划。
2. “实验历史”相关旧称在完整历史中保留，当前归档统一解释为“实验记录”体系。
3. “后续可扩展”“允许变化”类内容不属于 1.0.0 未完成需求，应进入后续版本设计。
4. 验证命令里记录的当时局部失败或非阻塞警告保留在完整历史中作为历史证据，不代表当前封存失败。
5. 与当前设计冲突的安装拓扑不在主入口中复述，只在 `INSTALL.md`、`.memory/design.md` 的禁止边界和本归档说明中保留。

## 重复、模糊和归类问题

1. 报告、实验历史、实验记录相关条目跨 2026-06-26、06-28、06-29、06-30 多次演进，单读 changelog 会感觉重复；归档中按“报告与历史”归类为一条产品线。
2. 工作流身份、工作流块、实验地图和重命名互相耦合，changelog 按日期拆开是合理的；归档中按“不可变定义 + 找回/探索”统一解释。
3. Furnace 历史有两次改动：先从表格转曲线和活动概览，再把概览改为后端聚合；归档中按最终状态只保留“聚合概览 + 日内曲线 + 导出”。
4. 前端 BEM、组件目录、浮层、通信入口分属多个条目；归档中统一为“前端基础设施收敛”。
