# MFC 流量计功能实现报告

**生成时间**: 2025-10-21
**版本**: v1.0
**范围**: 前端 → 后端 → 设备层完整功能链路

## 1. 整体架构概览

MFC功能采用三层架构设计，实现了完整的设备控制和数据监控：

```
前端 (React) ←→ 后端 (NestJS) ←→ 设备层 (FastAPI+Python) ←→ 物理设备
```

## 2. 前端实现

### 2.1 核心组件
- **`MFCDeviceCard.tsx`**: 设备卡片组件，提供完整的设备控制和状态显示界面
- **`mfcApi.ts`**: API封装类，实现所有MFC相关的HTTP请求

### 2.2 主要功能
- ✅ 设备扫描发现
- ✅ 实时状态监控（流量、设定值、百分比）
- ✅ 流量设定控制
- ✅ Hold/Follow模式切换
- ✅ 历史数据查询
- ✅ 错误处理和输入验证

### 2.3 UI特性
- 流量柱状图可视化显示
- 实时状态指示灯
- 百分比和sccm双单位显示
- 键盘快捷键支持（Enter确认）

## 3. 后端实现

### 3.1 控制器层 (`mfc.controller.ts`)
```typescript
/api/devices/mfc/scan      - POST - 设备扫描
/api/devices/mfc/devices   - GET  - 设备列表
/api/devices/mfc/status    - GET  - 状态查询
/api/devices/mfc/setpoint  - POST - 流量设定
/api/devices/mfc/logs/flow - GET  - 历史数据
```

### 3.2 服务层 (`mfc.service.ts`)
- 设备发现和缓存管理
- 状态轮询和数据处理
- FastAPI健康检查
- 错误处理和重试机制

### 3.3 设备通信层 (`mfc-device.service.ts`)
- FastAPI HTTP客户端封装
- 协议转换和数据映射
- 连接状态管理

## 4. 设备层实现

### 4.1 FastAPI服务器 (`mfc_device.py`)
- **串口通信**: 基于CS100系列MFC通信协议
- **协议解析**: 实现完整的命令帧构造和响应解析
- **设备扫描**: 自动地址扫描和设备信息获取
- **流量控制**: 支持sccm单位设定和百分比转换

### 4.2 核心功能
```python
scan()           # 设备扫描 (地址32-80)
read_status()    # 状态读取
write_setpoint_sccm() # 流量设定
write_hold_follow()    # 模式控制
```

### 4.3 通信协议
- **帧格式**: 地址 + 控制字节 + 长度 + 数据 + 校验和
- **数据类型**: UFRAC16百分比转换
- **错误处理**: 超时重试和校验和验证

## 5. 数据流分析

### 5.1 设备发现流程
```
前端调用scanDevices() → 后端FastAPI扫描 → 串口设备发现 → 返回设备列表
```

### 5.2 流量控制流程
```
用户输入sccm → 前端验证 → 后端转发 → Python协议转换 → 串口发送设备
```

### 5.3 状态监控流程
```
设备层轮询 → Python解析 → 后端缓存 → 前端实时更新
```
