前端端口选择后的数据流和通讯流分析
阶段1：端口选择（用户操作）
触发位置: MFCConnectionPanel.tsx:39
关键方法: handlePortSelection(port)
数据流:
用户从下拉列表选择端口
组件调用 mfcControls.connect(port, 19200, 1.0)
自动使用默认参数（波特率19200，超时1.0秒）
阶段2：自动连接（后台执行）
执行位置: useMfc.ts:295
关键方法: connect(port, baudrate, timeout)
数据流:
发送 API 请求到后端 POST /mfc/connect
更新前端状态为 connecting
连接成功后状态变为 connected
自动触发设备扫描（如果无设备）
阶段3：异步设备扫描（后台执行）
后端位置: mfc.service.ts:118
关键方法: scan(start, end)
数据流:
后端启动异步扫描（不阻塞HTTP响应）
扫描范围：地址32-80
实现"单地址发现即推送"模式
阶段4：实时WebSocket推送
后端位置: mfc.service.ts:193
关键方法: gateway.sendMfcDeviceDiscovered()
数据流:
每发现一个设备，立即通过WebSocket推送
推送数据：设备地址、气体类型、最大流量、连接状态
前端WebSocket服务接收并更新状态
阶段5：前端状态更新
前端位置: useMfc.ts:656
关键方法: handleDeviceDiscovered
数据流:
WebSocket回调更新 mfcState.devices
更新 lastUpdate 时间戳
UI自动刷新显示新设备
