# 故障排除指南

## 常见问题及解决方案

### 1. 连接线位置不正确

#### 症状描述
- 拖拽节点后，连接线没有跟随节点移动
- 连接线显示在错误的位置
- 连接线与节点没有正确连接

#### 可能原因
1. **算法不一致**：不同组件使用了不同的布局算法
2. **缓存问题**：布局计算缓存了过时的数据
3. **事件监听丢失**：连接线更新事件没有被正确处理

#### 解决方案

**检查算法一致性**
```typescript
// 确保所有地方都使用统一的服务
import { LayoutService } from '../services/layout';

const layout_service = LayoutService.getInstance();
const layout_params = layout_service.getDefaultLayoutParams(canvas_width, canvas_height);

// Canvas组件中使用
const position = layout_service.calculateNodePosition({
  index, nodes, layout_params
});

// Store中使用
const target_index = layout_service.calculateNodeIndex({
  position: drop_position,
  nodes, layout_params,
  allow_insert_at_end: true
});
```

**清除缓存**
```typescript
// 清除布局缓存，强制重新计算
layout_service.clearCache();

// 重新触发布局更新
connection_service.updateNodes(nodes, layout_params);
```

**检查事件监听**
```typescript
// 确保组件正确监听连接线更新事件
useEffect(() => {
  const handleConnectionPathsUpdated = (paths) => {
    setConnectionPaths(paths);
  };

  connection_service.on('connection-paths-updated', handleConnectionPathsUpdated);

  // 组件卸载时清理监听器
  return () => {
    connection_service.off('connection-paths-updated', handleConnectionPathsUpdated);
  };
}, [connection_service]);
```

### 2. 节点重叠或间距异常

#### 症状描述
- 节点之间出现重叠
- 节点间距过大或过小
- 节点超出画布边界

#### 可能原因
1. **画布尺寸不正确**：传入的画布宽度/高度有误
2. **节点宽度数据缺失**：节点没有正确的width属性
3. **布局参数配置错误**：padding或其他参数设置不当

#### 解决方案

**检查画布尺寸**
```typescript
// 确保传入正确的画布尺寸
const canvas_width = canvasRef.current?.clientWidth || 0;
const canvas_height = canvasRef.current?.clientHeight || 0;

if (canvas_width <= 0 || canvas_height <= 0) {
  console.warn('画布尺寸无效，使用默认值');
  return;
}

const layout_params = layout_service.getDefaultLayoutParams(canvas_width, canvas_height);
```

**验证节点数据**
```typescript
// 检查节点是否有正确的样式信息
const validated_nodes = nodes.map(node => ({
  ...node,
  style: {
    width: node.style?.width || 140,  // 默认宽度
    height: node.style?.height || 60, // 默认高度
    ...node.style
  }
}));
```

**调整布局参数**
```typescript
// 根据需要调整布局参数
const custom_layout_params = {
  canvas_width: 1200,
  canvas_height: 800,
  padding: 100,        // 增加边距避免节点贴边
  top_padding: 100,     // 顶部留白
  row_height: 150,      // 行高
  min_node_spacing: 60  // 最小间距
};
```

### 3. 拖拽后节点排列错乱

#### 症状描述
- 拖拽节点后，节点顺序发生意外变化
- 每行显示的节点数量发生变化
- 节点跳转到错误的位置

#### 可能原因
1. **索引计算错误**：拖拽目标位置计算不正确
2. **数组操作错误**：节点数组的splice操作有误
3. **布局重新计算时机问题**：布局更新的时机不正确

#### 解决方案

**调试索引计算**
```typescript
// 添加调试日志检查索引计算
const handleNodeDragEnd = (node, event) => {
  const drop_position = {
    x: event.clientX - rect.left,
    y: event.clientY - rect.top
  };

  console.log('拖拽位置:', drop_position);

  const target_index = layout_service.calculateNodeIndex({
    position: drop_position,
    nodes: current_nodes,
    layout_params,
    allow_insert_at_end: true
  });

  console.log('计算的目标索引:', target_index);
  console.log('当前节点索引:', current_nodes.findIndex(n => n.id === node.id));

  // 继续处理移动逻辑...
};
```

**验证数组操作**
```typescript
// 确保节点数组操作正确
const moveNode = (node_id: string, target_index: number) => {
  const current_index = nodes.findIndex(n => n.id === node_id);

  if (current_index === -1) {
    console.error('节点不存在:', node_id);
    return;
  }

  if (target_index === current_index) {
    console.log('目标位置相同，无需移动');
    return;
  }

  // 验证目标索引范围
  if (target_index < 0 || target_index > nodes.length) {
    console.warn('目标索引超出范围:', target_index);
    return;
  }

  const new_nodes = [...nodes];
  const [moved_node] = new_nodes.splice(current_index, 1);
  new_nodes.splice(target_index, 0, moved_node);

  console.log('节点移动结果:', {
    from: current_index,
    to: target_index,
    node_id: moved_node.id
  });

  return new_nodes;
};
```

**优化布局更新时机**
```typescript
// 使用防抖避免频繁的布局更新
const debouncedUpdateLayout = useMemo(
  () => debounce((nodes, layout_params) => {
    connection_service.updateNodes(nodes, layout_params);
  }, 100), // 100ms防抖
  []
);

// 在适当时机调用
useEffect(() => {
  if (canvasSize.width > 0 && nodes.length > 0) {
    debouncedUpdateLayout(nodes, layout_params);
  }
}, [nodes, canvasSize, debouncedUpdateLayout]);
```

### 4. 性能问题

#### 症状描述
- 布局计算导致页面卡顿
- 拖拽操作响应缓慢
- 大量节点时性能下降明显

#### 可能原因
1. **频繁的布局计算**：没有缓存机制，重复计算
2. **不必要的重新渲染**：组件更新过于频繁
3. **大量DOM操作**：连接线渲染效率低

#### 解决方案

**启用缓存**
```typescript
// 布局服务已经内置缓存机制，确保正确使用
const layout_service = LayoutService.getInstance();

// 避免清除缓存，除非必要
// layout_service.clearCache(); // 只在必要时调用

// 使用相同的layout_params以利用缓存
const layout_params = layout_service.getDefaultLayoutParams(canvas_width, canvas_height);
```

**优化渲染性能**
```typescript
// 使用React.memo优化连接线组件
const ConnectionLine = React.memo(({ path }: { path: ConnectionPath }) => {
  // 渲染逻辑
});

// 使用useMemo缓存计算结果
const connection_paths = useMemo(() => {
  return connection_service.getConnectionPaths();
}, [nodes, layoutStable]);

// 使用虚拟化处理大量连接线
const visible_connections = useMemo(() => {
  return connection_paths.slice(0, MAX_VISIBLE_CONNECTIONS);
}, [connection_paths]);
```

**减少DOM操作**
```typescript
// 批量更新节点位置
const batchUpdatePositions = (position_updates: Array<{index: number, position: Position}>) => {
  const updated_nodes = [...nodes];

  position_updates.forEach(({index, position}) => {
    if (updated_nodes[index]) {
      updated_nodes[index] = {
        ...updated_nodes[index],
        position
      };
    }
  });

  // 一次性更新所有节点
  setNodes(updated_nodes);
};
```

### 5. TypeScript类型错误

#### 症状描述
- 编译时出现类型错误
- API调用时类型不匹配
- 属性访问时报错

#### 可能原因
1. **类型定义不完整**：缺少必要的类型定义
2. **导入路径错误**：类型文件导入路径不正确
3. **snake_case命名不一致**：参数命名不符合规范

#### 解决方案

**检查类型导入**
```typescript
// 确保正确导入类型
import type {
  Position,
  LayoutParameters,
  DynamicLayoutConfig,
  ConnectionPath,
  ElectrochemicalNode
} from '../services/layout/types';

import { LayoutService, ConnectionBindingService } from '../services/layout';
```

**验证snake_case命名**
```typescript
// 确保所有参数使用snake_case命名
const layout_params: LayoutParameters = {
  canvas_width: 1200,    // ✅ 正确
  canvas_height: 800,    // ✅ 正确
  // padding: 100,       // ✅ 正确
  // topPadding: 100,     // ❌ 错误，应该是top_padding
};

const node_position: Position = {
  x: 100,
  y: 200  // ✅ 正确
};
```

**添加类型断言**
```typescript
// 在必要时使用类型断言
const response = await api.get('/nodes') as any;
const nodes = response.data as ElectrochemicalNode[];
```

## 预防性措施

### 1. 代码审查检查清单

在提交代码前，检查以下项目：

- [ ] 所有布局相关的调用都使用了统一的服务
- [ ] 参数命名遵循snake_case规范
- [ ] 事件监听器在组件卸载时正确清理
- [ ] 布局参数经过验证（画布尺寸 > 0）
- [ ] 节点数据包含必要的width和height属性
- [ ] 没有直接的DOM操作，都通过React状态更新
