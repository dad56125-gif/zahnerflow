# 迁移指南

## 概述

本指南帮助开发者从旧的布局系统迁移到新的统一布局服务。迁移过程需要修改多个文件，但总体来说是渐进式的，不会破坏现有功能。

## 迁移前准备

### 1. 备份当前代码

```bash
# 创建迁移前的备份分支
git checkout -b backup/layout-system-before-migration
git add .
git commit -m "backup: 布局系统重构前的完整备份"

# 切换回主分支
git checkout main
```

### 2. 理解现有架构

在开始迁移前，请仔细阅读以下文件，理解当前的实现：
- `apps/frontend/src/components/Canvas.tsx` - 主要的布局计算逻辑
- `apps/frontend/src/components/ConnectionLines.tsx` - 连接线渲染逻辑
- `apps/frontend/src/stores/canvasStore.ts` - 节点管理和拖拽逻辑

找到并删除以下函数：

- `calculateNodePosition`

替换为服务调用：

```typescript
export const Canvas: React.FC<CanvasProps> = ({
  zoomLevel,
  selectedWorkstation,
  onZoomIn,
  onZoomOut,
  onResetZoom,
  showWorkflowManager = false,
  onToggleWorkflowManager,
  showFilePathManager = false,
  onLoopDetected
}) => {
  // 获取服务实例
  const layout_service = LayoutService.getInstance();
  const connection_service = ConnectionBindingService.getInstance();

  // 使用统一的布局参数
  const layout_params = layout_service.getDefaultLayoutParams(
    canvasSize.width,
    canvasSize.height
  );

  // 监听连接线服务事件
  useEffect(() => {
    const handleLayoutRecalculated = (positions: Map<number, Position>) => {
      // 更新节点位置
      const updated_nodes = nodes.map((node, index) => {
        const position = positions.get(index);
        return position ? { ...node, position } : node;
      });
      setNodes(updated_nodes);
    };

    const handleConnectionPathsUpdated = (paths: ConnectionPath[]) => {
      setConnections(paths);
    };

    connection_service.on('layout-recalculated', handleLayoutRecalculated);
    connection_service.on('connection-paths-updated', handleConnectionPathsUpdated);

    return () => {
      connection_service.off('layout-recalculated', handleLayoutRecalculated);
      connection_service.off('connection-paths-updated', handleConnectionPathsUpdated);
    };
  }, [nodes, connection_service]);

  // 当画布尺寸或节点变化时，重新计算布局
  useEffect(() => {
    if (canvasSize.width > 0 && nodes.length > 0) {
      connection_service.updateNodes(nodes, layout_params);
    }
  }, [canvasSize.width, nodes.length, connection_service, layout_params]);
```

export const ConnectionLines: React.FC = () => {

        <ConnectionLinePath

// 单个连接线组件
const ConnectionLinePath: React.FC<{ path: ConnectionPath; index: number }> = ({ path }) => {
  const {
    start_position,
    end_position,
    path_points,
    connection_type
  } = path;

  if (connection_type === 'straight') {
    return (
      <line
        x1={start_position.x}
        y1={start_position.y}
        x2={end_position.x}
        y2={end_position.y}
        className="connection-line"
        stroke="rgba(255,255,255,0.6)"
        strokeWidth="2"
        markerEnd="url(#arrowhead)"
      />
    );
  } else {
    const path_data = `M ${start_position.x} ${start_position.y} ` +
      path_points.slice(1, -1).map(p => `L ${p.x} ${p.y}`).join(' ') +
      ` L ${end_position.x} ${end_position.y}`;

    return (
      <path
        d={path_data}
        className="connection-line curved"
        fill="none"
        stroke="rgba(255,255,255,0.6)"
        strokeWidth="2"

### 步骤 4: 更新Canvas Store

#### 4.1 修改store实现

在 `apps/frontend/src/stores/canvasStore.ts` 中：

```typescript
import { LayoutService } from '../services/layout';
import { ConnectionBindingService } from '../services/layout';

export const useCanvasStore = create<CanvasStore>((set, get) => {
  const layout_service = LayoutService.getInstance();
  const connection_service = ConnectionBindingService.getInstance();

  return {
    // ... 其他属性保持不变

    // 重构后的moveNode方法
    moveNode: (nodeId: string, newPosition: Position) => {
      const { nodes, canvasSize } = get();

      const node_index = nodes.findIndex(node => node.id === nodeId);
      if (node_index === -1) return;

      // 使用统一的索引计算
      const layout_params = layout_service.getDefaultLayoutParams(
        canvasSize.width,
        canvasSize.height
      );

      const target_index = layout_service.calculateNodeIndex({
        position: newPosition,
        nodes,
        layout_params,
        allow_insert_at_end: true
      });

      if (target_index === node_index) return;

      // 重新排列节点数组
      const new_nodes = [...nodes];
      const [moved_node] = new_nodes.splice(node_index, 1);
      new_nodes.splice(target_index, 0, moved_node);

      // 使用统一的位置计算
      const repositioned_nodes = new_nodes.map((node, i) => {
        const position = layout_service.calculateNodePosition({
          index: i,
          nodes: new_nodes,
          layout_params
        });
        return { ...node, position };
      });

      // 更新状态
      set({
        nodes: repositioned_nodes,
        validationError: validateNodes(repositioned_nodes)
      });

      // 通知连接线服务更新
      connection_service.updateNodes(repositioned_nodes, layout_params);
    },

    // 新增：统一的位置更新方法
    recalculatePositions: () => {
      const { nodes, canvasSize } = get();
      const layout_params = layout_service.getDefaultLayoutParams(
        canvasSize.width,
        canvasSize.height
      );

      const repositioned_nodes = nodes.map((node, i) => {
        const position = layout_service.calculateNodePosition({
          index: i,
          nodes,
          layout_params
        });
        return { ...node, position };
      });

      set({ nodes: repositioned_nodes });
      connection_service.updateNodes(repositioned_nodes, layout_params);
    }
  };
});
```

## 常见迁移问题

### 1. 类型错误

**问题**: TypeScript编译出现类型不匹配错误

**解决方案**:
- 检查是否正确导入了类型定义
- 确保参数命名使用snake_case
- 验证API调用参数类型正确

### 2. 事件监听问题

**问题**: 连接线不更新或更新延迟

**解决方案**:
- 确保正确注册了事件监听器
- 检查组件卸载时是否清理了监听器
- 验证事件触发时机是否正确

### 3. 性能问题

**问题**: 布局计算导致性能下降

**解决方案**:
- 检查是否意外清除了缓存
- 确保使用防抖处理频繁更新
- 优化组件渲染，使用React.memo

### 4. 布局异常

**问题**: 节点位置计算不正确

**解决方案**:
- 验证传入的画布尺寸是否正确
- 检查节点数据是否包含必要的width/height属性
- 确认布局参数配置合理
