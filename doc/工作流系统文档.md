# ZahnerFlow 工作流系统文档

## 概述

ZahnerFlow 工作流系统是一个基于 NestJS 后端和 TypeScript 前端的完整工作流管理平台。系统采用 pnpm monorepo 架构，通过 @packages/types 实现前后端类型共享，确保接口一致性。

## 系统架构

### 整体架构
```
┌─────────────────┐    API调用    ┌─────────────────┐
│   前端应用      │◄────────────►│   后端服务      │
│                 │               │                 │
│ - React/Vue     │               │ - NestJS        │
│ - TypeScript    │               │ - TypeScript    │
│ - workflowService│              │ - workflow.module│
└─────────────────┘               └─────────────────┘
         │                                 │
         │              ┌─────────────────┐
         └─────────────►│   共享类型      │
                        │                 │
                        │ - @packages/types│
                        │ - 统一接口定义   │
                        └─────────────────┘
```

### 架构原则
1. **后端作为状态机核心**：定义状态流转规则和业务逻辑
2. **前端只负责展示和操作**：通过 API 调用后端，根据返回状态更新 UI
3. **类型约束**：使用 @packages/types 确保前后端接口一致性
4. **模块化设计**：清晰的职责分离，便于维护和扩展

### 状态流转规则

#### 工作流生命周期
1. **创建阶段**：生成唯一ID → 验证定义 → 持久化存储
2. **编辑阶段**：验证修改 → 版本递增 → 更新存储
3. **执行阶段**：状态更新 → 进度跟踪 → 结果记录
4. **归档阶段**：数据备份 → 清理缓存 → 历史记录

#### 验证机制
- **基本信息验证**：名称、描述必填检查
- **节点验证**：ID唯一性、类型有效性、配置完整性
- **边验证**：源/目标节点存在性、循环检测
- **连通性检查**：确保工作流图的完整性

### 前端组件集成

#### 使用示例
```typescript
// 创建工作流
const workflow = await workflowService.createWorkflow({
  name: '测试工作流',
  description: '这是一个测试工作流',
  workstation: 'WS001',
  definition: {
    id: 'def-1',
    name: '测试定义',
    description: '测试工作流定义',
    nodes: [...],
    edges: [...],
    version: 1
  }
});

// 执行工作流
const execution = await executionService.executeWorkflow(
  workflow.id,
  {
    parameters: { param1: 'value1' },
    priority: 'normal'
  }
);

// 监控执行状态
const status = await executionService.getExecutionStatus(execution.executionId);
```
