## 重构规则
1. **必须按照frontendrebuilt.md的架构执行**：严格按照frontendrebuilt.md文档中定义的新架构进行重构，不得偏离既定结构
2. 每次重构一个模块，迁移一个模块的所有文件
3. 模块内文件导入迁移（工具：仅允许使用edit）
4. 重新审查此重构后的模块，判断是否有问题，如有，解决，禁止简化，禁止更改文件内容，只允许解决导入问题
5. 回到Todo.md 更新Todo.md 然后执行下一个模块的修改

## 注意事项
1. **禁止使用简化操作**：所有导入路径修复必须使用Edit工具逐个修改
2. **禁止更改文件内容**：除了导入路径外，不能修改任何其他代码内容
3. **只解决导入问题**：重构的目标是解决导入路径问题，不是功能改进
4. **逐步执行**：完成一个模块的所有步骤后再进行下一个模块
5. **及时更新状态**：每完成一个步骤都要更新此Todo.md文档

## 重构顺序
1. shared -> canvas -> layout -> panels -> workflow -> loop -> mfc -> furnace -> user -> core -> app -> main.tsx
2. 从共享模块开始，逐步向依赖模块推进
3. 最后处理根级别的应用入口文件

## 子代理执行规则

### 总代理职责
1. **管控所有任务的流程**：负责协调整个重构过程
2. **禁止执行任何单项任务**：只负责监督和决策
3. **启动和监督子代理**：为每个具体任务启动相应的子代理
4. **审查子代理结果**：分析子代理的报告并做出决策
5. **维护Todo状态**：及时更新任务进度

### 子代理执行要求
1. **专门化执行**：每个子代理专注于特定任务
2. **详细报告**：提供完整的执行过程和结果报告
3. **问题识别**：准确识别所有遇到的问题
4. **解决方案建议**：为发现的问题提供解决方案

### 强制执行规则
1. **禁止退出**：只要有启动问题就必须解决，不允许中断
2. **完全修复**：所有编译和运行时错误都必须解决
3. **验证完成**：必须验证应用能正常启动和运行
4. **文档更新**：完成所有任务后必须更新Todo状态
