# 点击外部关闭功能 - 测试清单

## 测试目的

验证所有应用了 `useOnClickOutside` Hook 的组件都能正确响应点击外部区域关闭的操作。

## 测试环境

- 浏览器：Chrome/Firefox/Safari/Edge
- 设备：桌面/触摸设备（移动设备/平板）

## 测试用例

### 1. 工作流管理面板

**操作步骤**：
1. 点击工具栏中的"📋 工作流"按钮打开面板
2. 确认面板出现在屏幕中央
3. 点击面板外的任意区域
4. 观察面板是否自动关闭

**预期结果**：✅ 点击外部区域后，面板立即关闭

**失效情况**：❌ 面板未关闭，或需要点击关闭按钮才关闭

## 自动化测试建议

如果您使用测试框架（如 Jest + Testing Library），可以编写以下测试：

```typescript
// 示例测试代码
describe('useOnClickOutside', () => {
  test('点击外部区域应调用关闭回调', () => {
    const handleClose = jest.fn();
    render(<TestComponent onClose={handleClose} />);

    // 点击外部区域
    fireEvent.mouseDown(document);

    expect(handleClose).toHaveBeenCalledTimes(1);
  });

  test('点击内部区域不应调用关闭回调', () => {
    const handleClose = jest.fn();
    render(<TestComponent onClose={handleClose} />);

    // 点击内部区域
    const internalElement = screen.getByTestId('internal');
    fireEvent.mouseDown(internalElement);

    expect(handleClose).not.toHaveBeenCalled();
  });
});
```

## 已知问题

**如果发现任何问题，请检查**：

1. **Hook 导入路径是否正确**
   ```typescript
   import { useOnClickOutside } from '../services/hooks/useOnClickOutside';
   ```

2. **ref 是否正确附加**
   ```typescript
   const ref = useRef<HTMLDivElement>(null);
   // JSX
   <div ref={ref}>...</div>
   ```

3. **enabled 参数是否正确设置**
   ```typescript
   // 只有在组件显示时才启用监听
   useOnClickOutside(ref, handler, isOpen);
   ```
